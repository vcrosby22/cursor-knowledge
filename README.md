## How to use this

1. **Star / fork / clone** if you want a starting point for your own knowledge log.
2. In your **private** workspace, keep journals, profiles, `.env`, and anything personal **out** of this public repo.
3. Optionally **copy updates** from your private `cursor-knowledge/CURSOR_KNOWLEDGE.md` into this file when you want to publish new learnings.
4. Replace or extend **`PROJECT_CONTEXT.md`** with your own goals in private; the file here is intentionally generic.
